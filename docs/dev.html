Development

Tethys is developed on the branch dev and merged to the master with each new version. Together with a note in the release notes. Before contributing please read the concept and development principles carefully.

Development principles

Please read Tethys' concept. From version 1.0 every new version has to be downward compatible.

Code style

* Classnames begin with an uppercase letter.

Release

Before merging the dev branch onto the master: * Check for TODOs * Check for out commented, unused and deprecated code * Check code format, check compiler warnings * Check comments (translate into english) and add comments * Update and add documentation for new content * Write release note * Create tag
